# PawPal+ Agentic Workflow — Implementation Checklist

##Extension being Added:
Integrate agentic planning and error-logging into the existing workflow
- Agentic loop + logging
## AI Features Being Added
- Rule-based AI agent (PawAgent) that autonomously manages pet care schedules
- Full schedule generation for new/empty pets using species + age + breed templates
- Urgency escalation: overdue tasks get priority boosted automatically
- Coverage checking: missing feeding/activity/hygiene tasks are auto-added
- Conflict resolution: clashing time slots are automatically rescheduled
- Species/age rules: senior dogs get vet checks, cats get litter tasks, etc.
- Visible reasoning log: every agent decision is shown step-by-step in the UI

## System Diagram

The system diagram is separate from the existing class diagram (final_UML.txt).
It shows data flow, main components, and where humans/tests are involved.

To be written as a Mermaid flowchart in a new file: system_diagram.txt

### Components to include:
- User (input)
- Streamlit UI
- Owner / Pet / Task objects
- PawAgent (the agent)
- Rules Engine (Profile, Urgency, Coverage, Species, Conflict, Guardrails)
- AgentDecision list (output of agent)
- Reasoning Log + Updated Schedule (UI output)
- Human review (user reads reasoning log)
- pytest (tests verify agent behavior)

### Data flow:
[User: enters pet info + tasks]
            ↓
    [Streamlit UI]
            ↓
  [Owner / Pet / Task objects]
            ↓
     [PawAgent.run()]
            ↓
  ┌─── Rules Engine ────┐
  │  Profile Rule       │
  │  Urgency Rule       │
  │  Coverage Rule      │
  │  Species Rule       │
  │  Conflict Rule      │
  │  + Guardrails       │
  └─────────────────────┘
            ↓
  [list of AgentDecisions]
            ↓
  [Streamlit UI — Reasoning Log + Updated Schedule]
            ↓
  [Human reviews decisions] + [pytest verifies behavior]
